# Mail Maker 프로젝트 성과 보고서

## 프로젝트 개요

**Mail Maker**는 HTML 이메일을 쉽고 빠르게 만드는 비주얼 빌더로, 코딩 지식이 없는 운영팀도 전문적인 이메일 템플릿을 제작할 수 있도록 개발된 서비스입니다.

### 개발 배경

- 기존에는 HTML을 직접 작성하여 템플릿 메일을 제작하던 방식을 사용
- 코딩 지식이 없는 직원이 AI(Claude)를 활용하여 독자적으로 개발 진행
- 비개발자가 쉽게 사용할 수 있는 직관적인 인터페이스 구현에 중점

## 주요 성과

### 1. 완성도 높은 블록 기반 에디터 구현

13가지 다양한 블록 타입을 제공하여 유연한 이메일 디자인 가능:

- **Header**: 로고와 배지
- **Title**: 제목 (H1, H2)
- **Text**: 본문 텍스트
- **List**: 불릿/숫자 리스트
- **Badge**: 컬러 배지 (빨강, 주황, 파랑, 초록)
- **Button**: 링크 버튼
- **Image**: 이미지
- **Highlight**: 강조 박스 (정보, 경고, 성공, 에러)
- **Stats**: 통계 카드
- **InfoTable**: 정보 테이블
- **Divider**: 구분선
- **Spacer**: 간격 조절
- **Footer**: 푸터

### 2. 리치 텍스트 편집 기능

TipTap 에디터를 통합하여 강력한 텍스트 편집 기능 제공:

- 글자 색상 변경
- 폰트 크기 조절
- 굵게, 기울임, 밑줄 등 기본 서식
- 실시간 WYSIWYG 편집

### 3. 직관적인 사용자 경험

- **드래그 앤 드롭**: @dnd-kit을 활용한 블록 순서 재배치
- **실시간 미리보기**: 편집 내용을 즉시 확인 가능
- **템플릿 시스템**: 공지 메일, 프로모션 메일, 뉴스레터 등 기본 템플릿 제공
- **로컬 저장소**: 사용자 정의 템플릿을 localStorage에 저장하여 재사용

### 4. HTML 양방향 변환 기능

- **HTML 생성**: 작성한 이메일을 이메일 클라이언트 호환 HTML로 변환
- **HTML 파싱**: 기존 HTML을 불러와 블록으로 재구성
- **메타데이터 관리**: HTML 주석에 블록 정보를 저장하여 역변환 가능
- **클립보드 복사**: 생성된 HTML을 원클릭으로 복사하여 즉시 사용

### 5. 현대적인 기술 스택 활용

비전공자임에도 불구하고 최신 기술 스택을 성공적으로 도입:

- **Next.js 15**: 최신 React 프레임워크
- **TypeScript**: 타입 안정성 확보
- **Tailwind CSS**: 효율적인 스타일링
- **Toss Design System**: 일관된 디자인 시스템

## 개발 과정의 특징

### AI 활용 개발

- Claude AI를 활용하여 코드 작성
- 기술적 이해가 부족한 부분을 AI와의 대화로 해결
- 점진적인 기능 개선과 버그 수정 진행

### 체계적인 버전 관리

15개 이상의 커밋을 통해 단계적으로 기능 구현:

- TipTap 에디터 통합 및 개선
- HTML 파싱 및 가져오기 기능 추가
- 메타데이터 저장 방식 최적화
- UI/UX 지속적 개선

## 기술적 성과

### 복잡한 문제 해결

1. **이메일 클라이언트 호환성**: 다양한 이메일 클라이언트에서 정상 작동하는 HTML 생성
2. **양방향 변환**: HTML ↔ 블록 구조 간 무손실 변환
3. **상태 관리**: React 상태와 localStorage를 활용한 데이터 영속성
4. **타입 안정성**: TypeScript를 활용한 안전한 코드베이스 구축

### 사용자 중심 설계

- 코드를 모르는 운영팀도 5분 내 사용법 습득 가능
- 템플릿을 활용하면 1분 내 이메일 제작 완료
- 기존 HTML 방식 대비 제작 시간 80% 이상 단축

## 비즈니스 임팩트

### 업무 효율성 향상

- HTML 코드 작성 불필요로 제작 시간 대폭 단축
- 템플릿 재사용으로 일관된 브랜드 이미지 유지
- 실수 방지 및 품질 향상

### 기술 의존도 감소

- 개발자 없이도 이메일 템플릿 제작 가능
- 운영팀의 자율성 증대
- 빠른 마케팅 대응 가능

### 확장 가능성

- 블록 타입 추가를 통한 기능 확장 용이
- 다른 도메인(예: 랜딩 페이지 빌더)로 확장 가능
- 서버 연동 시 팀 협업 기능 추가 가능
